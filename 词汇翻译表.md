## 4. 词法细节

### 4.1 标识符 `IDENT`

```
[_A-Za-z][_A-Za-z0-9]*
```

- 符合 C 标识符规范。
- 匹配后查关键字表；命中则返回对应 `KW_*`，否则返回 `IDENT`。

### 4.2 整数 `NUMBER`

```
-?(0|[1-9][0-9]*)
```

**负号消歧规则**（避免 `a-5` 被拆成 `a` + `NUMBER(-5)`）：

| 前一个 Token 类型                              | 见到 `-` 后的行为        |
| ---------------------------------------------- | ------------------------ |
| `IDENT` / `NUMBER` / `RPAREN`                  | 输出 `MINUS`，再读数字   |
| 其他（文件开头、运算符、`(`、`,`、关键字等）   | 将 `-` 并入数字字面量    |

示例：

| 源码片段      | 词法序列                                      |
| ------------- | --------------------------------------------- |
| `return -5;`  | `return` `NUMBER(-5)` `;`                   |
| `a - 5`       | `a` `MINUS` `NUMBER(5)`                     |
| `a-5`         | `a` `MINUS` `NUMBER(5)`                     |
| `(-1)`        | `(` `NUMBER(-1)` `)`                        |

### 4.3 空白与注释（跳过，不产生 Token）

| 类型       | 规则                                       |
| ---------- | ------------------------------------------ |
| 空白       | 空格、`\t`、`\r`、`\n`                     |
| 单行注释   | `//` … 到行末（不含换行）                  |
| 多行注释   | `/*` … `*/`（支持跨行；未闭合则词法错误）  |

### 4.4 双字符运算符优先级

扫描时优先尝试双字符，再回退单字符：

```
||  &&  <=  >=  ==  !=
```

| `"break"`          | `KW_BREAK`   |
| `"continue"`       | `KW_CONTINUE`|
| `"return"`         | `KW_RETURN`  |
| `"="`              | `ASSIGN`     |
| `"\|\|"`           | `OR`         |
| `"&&"`             | `AND`        |

| `","`              | `COMMA`      |
| `ID`               | `IDENT`      |
| `NUMBER`           | `NUMBER`     |
