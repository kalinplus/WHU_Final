# ToyC 语义分析开发计划

## Summary

为项目补齐独立 Sema 模块：符号表、语义检查、常量求值、`SemaResult` side-table，并把 `-dump-ir` 改为消费 Sema 结果。

## Key Changes

- 新增 `include/toyc/sema.h` 与 `src/sema/` 实现，暴露 `SymbolRef`、`ExprValueType`、`SemaResult`、`analyze()`。
- 将 `ast_contract.h::ValueKind` 改名为 `ValueType`，解除与 IR `ValueKind` 的命名冲突。
- 语义检查覆盖：声明/使用顺序、重定义、main 签名、函数调用、const 求值、void 误用、return 路径、break/continue 上下文等。
- `-dump-ir` 进入 IRGen 前先执行 Sema；IRGen 消费 `SemaResult` side-table，不再重复做语义解析。

## Test Plan

- 新增 `sema_regression` CTest，覆盖 valid/invalid 语义用例。
- 保持 parser、IR、IRGen 回归通过。
- 测试命令：

```powershell
cmake -S . -B build
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

## Assumptions

- 不修改 AST 节点，采用 side-table。
- 全局变量初值必须编译期可求值；局部变量初值允许运行期表达式。
- `-dump-ast` 保持前端专用，不强制 Sema。
